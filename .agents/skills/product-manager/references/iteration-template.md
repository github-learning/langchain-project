# 迭代规划模板

## 任务拆解原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **原子化** | 每个任务 0.5-2 天 | "登录功能" → "登录页面" + "登录API" + "登录测试" |
| **可验证** | 每个任务有明确完成标准 | 不是"完成某模块"而是"API返回正确" |
| **无遗漏** | 覆盖功能/测试/文档/数据 | 不要只拆开发任务 |
| **有依赖** | 明确标记任务依赖 | "登录API" 依赖 "用户表" |

## 迭代计划模板

```markdown
## 迭代计划 ITER-XXX

### 迭代目标
[本次迭代要达成什么]

### 迭代范围
| 模块 | 功能点 | 优先级 |
|------|-------|-------|
| 模块A | 功能1 | P0 |
| 模块A | 功能2 | P1 |

### 任务拆解
| 任务ID | 任务 | 负责 | 预计 | 依赖 | 验收标准 |
|--------|------|------|------|------|---------|
| T-001 | 开发模块A | dev | 2d | - | [标准] |
| T-002 | 开发模块B | dev | 3d | T-001 | [标准] |
| T-003 | 测试模块A | qa | 1d | T-001 | [标准] |
| T-004 | 准备测试数据 | kb | 0.5d | - | [标准] |

### 里程碑
| 日期 | 里程碑 | 交付物 |
|------|-------|-------|
| D+2 | Phase 1 完成 | 代码 + 单元测试 |
| D+5 | Phase 2 完成 | 代码 + 集成测试 |
| D+6 | 测试完成 | 测试报告 |
| D+7 | 验收完成 | 验收签字 |

### 风险预留
- Buffer: 20% 时间用于修复和调整
```
