# 需求文档模板

## 需求记录

```markdown
## 需求记录 REQ-XXX

### 原始需求
[用户/业务方的原始描述]

### 需求来源
- 提出人：
- 部门：
- 场景：

### 接收时间
YYYY-MM-DD
```

## 需求澄清检查表

| 问题 | 目的 |
|------|------|
| **背景** | 为什么需要这个功能？解决什么问题？ |
| **用户** | 谁会使用这个功能？使用频率？ |
| **场景** | 在什么情况下使用？ |
| **预期** | 你期望的结果是什么？ |
| **约束** | 有什么限制？时间/技术/资源？ |
| **验收** | 怎么算成功？怎么衡量？ |

## 完整需求文档模板

```markdown
## 需求文档 REQ-XXX

### 1. 需求概述

**背景**
[为什么要做这件事]

**目标**
[做完之后能达到什么效果]

**成功指标**
| 指标 | 当前值 | 目标值 | 测量方式 |
|------|-------|-------|---------|
| [指标1] | X | Y | [测量方法] |
| [指标2] | X | Y | [测量方法] |

### 2. 功能需求

**功能点 1**
- 描述：[做什么]
- 用户故事：作为 [用户]，我希望 [功能]，以便 [目的]
- 验收标准：
  - [ ] [可检查的条件]
  - [ ] [可检查的条件]

**功能点 2**
- ...

### 3. 非功能需求

| 类型 | 要求 |
|------|------|
| **性能** | 响应时间 < X ms |
| **可用性** | 可用性 > X% |
| **安全** | 符合 XXX 安全标准 |

### 4. 约束条件

| 类型 | 约束 |
|------|------|
| **技术** | [技术限制] |
| **业务** | [业务限制] |
| **时间** | [交付时间要求] |

### 5. 风险评估

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| [风险1] | [影响程度] | [高/中/低] | [应对措施] |

### 6. 附录

- 原始需求截图
- 相关文档链接
- 讨论记录
```
